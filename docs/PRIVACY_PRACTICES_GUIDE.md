# Chrome 应用商店 - 隐私惯例填写指南

根据您提供的截图和插件的代码逻辑 (`n8n Trigger`)，您需要在“隐私惯例” (Privacy practices) 页面勾选以下选项。

请注意：Google 的定义中，“收集”也包括扩展程序**访问**并**发送**数据（即使是发送给 OpenAI 或用户自己的 n8n，而不是发给您的服务器，也算作“收集/使用”）。

---

## 需要勾选的项目

请勾选以下 **4 项**：

### 1. ✅ 身份验证信息 (Authentication information)
*   **包含内容**：API 密钥 (OpenAI/ZhipuAI Key)、n8n 的用户名/密码。
*   **勾选理由**：您的插件需要保存这些凭证来连接第三方服务。
*   **后续问题回答**：
    *   **是否将此数据用于单一用途以外的目的？** 否 (No)
    *   **是否会传输此数据？** 是 (Yes) -> 因为要发给 LLM 接口验证。

### 2. ✅ 个人通讯 (Personal communications)
*   **包含内容**：电子邮件、短信或**聊天消息**。
*   **勾选理由**：您的插件核心功能是“聊天”，用户输入的 Prompt 和 AI 的回复属于此类别。
*   **后续问题回答**：
    *   **用途**：应用功能 (App functionality)

### 3. ✅ 网络记录 (Web history)
*   **包含内容**：用户访问过的网页及相关数据（如**网页标题**和访问时间）。
*   **勾选理由**：插件具有 `activeTab` 权限，会读取当前页面的 **URL** 和 **标题** (`document.title`, `window.location.href`) 并发送给 n8n 或作为 AI 上下文。
*   **后续问题回答**：
    *   **用途**：应用功能 (App functionality)

### 4. ✅ 网站内容 (Website content)
*   **包含内容**：文字、图片、声音、视频或超链接。
*   **勾选理由**：插件的“引用页面全文”和“选中文本”功能会直接读取网页上的**文字内容** (`document.body.innerText`, `window.getSelection()`)。
*   **后续问题回答**：
    *   **用途**：应用功能 (App functionality)

---

## 不需要勾选的项目

*   ⬜ **个人身份信息**：您的插件不要求用户注册，不收集姓名、地址、身份证号。
*   ⬜ **健康信息**：不涉及。
*   ⬜ **财务和付款信息**：不涉及。
*   ⬜ **位置**：不涉及 GPS 或 IP 定位。
*   ⬜ **用户活动**：虽然您监听点击，但通常这里指“网络监控、鼠标轨迹记录”等分析行为，您的插件只是响应操作，不记录行为日志。

---

## 常见后续问题参考

对于每一个勾选的项目，Google 可能会问以下三个问题，建议按如下回答：

1.  **您如何使用此数据？ (How do you use this data?)**
    *   选择：**应用功能 (App functionality)**。

2.  **您是否会传输此数据？ (Do you transmit this data?)**
    *   选择：**是 (Yes)**。
    *   *解释*：虽然不传给开发者（您），但数据会传给 **OpenAI/ZhipuAI** 或 **用户的 n8n 服务器**。在 Google 看来，只要离开浏览器就算传输。

3.  **您的服务器是否会处理此数据？**
    *   选择：**否 (No)**。（如果您没有中间服务器，全是直连的话）。
    *   *注意*：如果是直连 OpenAI，这里选“否”通常是可以的，或者选“是”但在描述中说明是传给第三方服务商。**最稳妥的策略是：** 诚实地表明数据离开了设备（Transfer），但在隐私政策中明确说明是直连第三方。
